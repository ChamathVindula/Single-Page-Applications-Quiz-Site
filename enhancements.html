Enhancement 1
Index Page Animation

  • I have applied a simple animation to show the background processes which take place when a user interacts with a single page application.
  • This involves the description of specific moments of the animation with the @keyframe rule. This allows you to have repeating animations that don't depend on user actions or JavaScript to get triggered.
  • This particular animation is done by inserting 14 individual images to the index page using html which represents each element in the diagram. Then each image is given specific properties and values in css to built up the initial basic diagrammatic layout (without animation).
  • Then the @keyframe rule is used to specify direction and distance the specific image is to travel.
  • Keyframes had to be created for each moving image as their sizes differed from one another therefore resulting in each image having to travel a different distance to make certain images move as a single unit.
  • Each keyframe identifier has to be then added to the specific image selector (in this case id) to create a link between the image and the animation.
  • Other animation related properties such as animation-duration, animation-delay, animation-iteration-count had to be also added to the images in css.
  • General keyframe rule applied was;
        
     @keyframes keyframe_identifier{
          from{transform: transform_property(value)}
          to{transform: transform_property(value)}
        }
  • General digram element properties used;
        
    #element_id{position:position_value;
          width: value;
          height: value;
          top: value;
          left: value;
          animation-name: keyframe_identifier;
          animation-duration: time_in_seconds;
          animation-delay: time_in_seconds;
          animation-iteration-count: value;
          animation-fill-mode: value;
       }
  • I have not used any third party code in this enhancement and is entirely my own code but I used a YouTube video to understand the concept of animations and how it works. The source is cited below:
